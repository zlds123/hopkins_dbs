Phase 2 feasibility, explained

A “pseudo-online replay” feeds the recording back in time order, as if it were arriving live, and asks three decoders to predict movement-vs-rest one block at a time. The only difference between the decoders is how they learn. Two experiments test (1) whether the data drifts on its own and (2) whether adaptation rescues accuracy when it does.

Read this first: AJILE12 here is a feasibility control, not the main result
AJILE12 has no stimulation and little within-session drift, so it cannot test online adaptation on its own. Its role is to prove the streaming machinery is correct. The real Phase 2 runs on the DARPA RAM dataset (ds005557), which has logged closed-loop stimulation and outcomes — see the last section.

The three decoders — the only difference is when they learn

DecoderWhen it learnsWhat it remembersAnalogy
static Once, on the warm-up. Then frozen. Only the warm-up period. Study for the test once, never review again.
online-SGD A small weight nudge after every block. All past blocks (fading memory). Review a little every day; old lessons slowly fade.
sliding-refit Throws away the old model; retrains from scratch on the last K blocks before each test. Only the most recent K blocks. Re-study from scratch using just this week’s notes.

online vs sliding: both adapt, but differently. online-SGD is incremental — it keeps one model alive forever and edits it gently, so it carries long memory but reacts slowly. sliding-refit is amnesiac — it rebuilds a brand-new model from only the recent window, so it reacts fast to change but forgets old structure.

Two experiments, told simply

1 · Observed (within-session) real data

Just replay ~75 minutes of one real recording and decode movement. Nothing is tampered with. Question: does the brain signal drift on its own within a session?

DecoderAUC
static (frozen)0.626
online-SGD0.631
sliding-refit0.649
null (shuffled)0.514

Static even improved over time (0.58 → 0.65). Adapting barely helped (+0.02).

Proves: movement is decodable, but within a single session the signal is basically stable — so adaptation isn’t really needed here. (This is the honest point that backs the mentors’ doubt.)
2 · Induced drift (rotation) positive control

Same data, but we deliberately and gradually scramble how the signal is laid out over time (a rotation that keeps the information but moves it around). Question: if drift does happen, does adapting rescue us?

DecoderAUC
static (frozen)0.533
online-SGD0.607
sliding-refit0.626
null (shuffled)0.514

Static collapsed toward chance (0.55 → 0.52); the adaptive decoders held ~0.61–0.63.

Proves: when drift reorganizes the signal, a frozen decoder breaks and the adaptive ones recover. The adaptation machinery works — the open question is whether real drift (e.g. across RAM’s logged sessions) is this strong.

Where Phase 2 actually runs: RAM (ds005557)

AJILE12 only lets a decoder watch behavior — it never acted on the brain. To study online adaptation of a real control loop, Phase 2 moves to the DARPA RAM closed-loop iEEG corpus, where a classifier actually decided when to stimulate and every action and outcome was logged. That lets the recorded outcomes be reused to score a new policy, with no synthetic brain model.

RAM as a logged decision loop
Loop pieceWhat it is in RAM

Off-policy action-matching

The candidate decoder proposes an action at each step. We trust the logged outcome only where its action matches what was actually delivered; where they diverge, the recorded future no longer applies.

Gating risk: the match rate
This only works if the agent and the logged policy agree often enough to leave a usable sample. The first step in Phase 2 is to measure that match rate before anything else — if it is too low, OPE is underpowered.
Translation target
The clinical endpoint is psychiatric DBS (e.g. the Provenza OCD and Summit RC+S programs on DABI), where drift spans days–weeks and stimulation is logged — but access is gated and raw audio/video is excluded, so RAM is the open bridge for the method.
The one-sentence takeaway
On AJILE12 the harness works and movement is decodable, but within-session drift is too small to need adaptation; the rotation control proves adaptation rescues accuracy when drift is real — so Phase 2 moves to RAM (ds005557), where logged closed-loop stimulation makes online adaptation and off-policy evaluation an honest test.
